## Problem Statement : 
Disaster situations like floods, earthquakes, and urban emergencies often face poor coordination and communication between citizens, government agencies, healthcare providers, and NGOs, which leads to delays, inefficient use of resources, and greater loss of life and property, highlighting the need for a centralized, role-based disaster management system.

## Project Overview :
* The system is based on a centralized Disaster Management Network that connects all parts of the system and helps in smooth coordination.
* It includes three main enterprises: Government Emergency Services, Healthcare Services, and NGO & Relief Services.
* Government Emergency Services handle police and fire-related emergency operations.
* Healthcare Services manage hospitals and ambulance support for medical needs.
* NGO & Relief Services take care of food supply and shelter for affected people.
* Each organization has specific roles like citizen, police, doctor, and volunteers to handle different tasks.
* Overall, the system is well-structured and helps different organizations work together efficiently during disasters.
* The system uses a multi-layered design where users like citizens, officials, doctors, and NGO workers interact through role-based dashboards.
* The application layer is the core part that handles login, request creation, assignment, tracking, and overall system logic.
* The system follows a clear structure where a network contains enterprises, enterprises contain organizations, and organizations manage users and tasks.
* The work request module helps in creating, assigning, tracking, and completing requests efficiently.
* The data layer stores all important information like users, roles, organizations, and requests.
* The authentication module ensures secure login and restricts access based on user roles.
* The reporting module helps in analyzing request status, response time, and system performance.
* The configuration module is used to add sample data for testing and demonstration.
